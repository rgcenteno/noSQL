# Comandos básicos de Redis

## `PING`

Comprueba que el servidor Redis está vivo. Responde con un `PONG`.

```redis
127.0.0.1:6379> PING
PONG
```

## `SET clave valor`

Guarda un par clave-valor en Redis.

```redis
127.0.0.1:6379> SET aitor-bd "url"
OK
```

## `GET clave`

Recupera el valor de una clave.

```redis
127.0.0.1:6379> GET aitor-bd
"aitor-medrano.github.io/bd"
```

## `MSET k1 v1 k2 v2 ...`

Inserta múltiples pares clave-valor.

```redis
127.0.0.1:6379> MSET a v1 b v2
OK
```

## `MGET k1 k2 ...`

Recupera varios valores a la vez.

```redis
127.0.0.1:6379> MGET a b
1) "v1"
2) "v2"
```

## `KEYS patrón`

Muestra todas las claves coincidentes. No se recomienda en producción.

```redis
127.0.0.1:6379> KEYS aitor-*
1) "aitor-bd"
```

## `SCAN cursor MATCH patrón`

Itera claves sin bloquear el servidor. Es una alternativa a `KEYS`.

```redis
127.0.0.1:6379> SCAN 0 MATCH usuario:*
1) "0"
2) 1) "usuario:1001"
   2) "usuario:1002"
```

## `SCAN cursor MATCH patrón COUNT número TYPE tipo`

Itera claves sin bloquear el servidor. `COUNT` indica una cantidad aproximada de elementos que Redis intentará devolver por iteración. `TYPE` filtra por tipo de dato.

```redis
127.0.0.1:6379> SCAN 0 MATCH usuario:* COUNT 3 TYPE "string"
1) "0"
2) 1) "usuario:1001"
   2) "usuario:1002"
   3) "usuario:1003"
```

## `DEL clave`

Elimina una clave y su valor.

```redis
127.0.0.1:6379> DEL aitor-bd
(integer) 1
```

## `EXPIRE clave segundos`

Establece un tiempo de vida, o TTL, para una clave.

```redis
127.0.0.1:6379> EXPIRE a 3600
(integer) 1
```

## `TTL clave`

Muestra el tiempo restante hasta la expiración de una clave.

```redis
127.0.0.1:6379> TTL a
(integer) 3283
```

## `INCR clave`

Incrementa un valor entero. Es útil para contadores.

```redis
127.0.0.1:6379> INCR contador
(integer) 3
```

## `MULTI` y `EXEC`

`MULTI` comienza una transacción y `EXEC` la confirma.

```redis
127.0.0.1:6379> MULTI
OK
127.0.0.1:6379> SET clave valor
QUEUED
127.0.0.1:6379> INCR contador
QUEUED
127.0.0.1:6379> EXEC
1) OK
2) (integer) 4
```

## `SAVE`

Fuerza un guardado de snapshot RDB en disco de forma bloqueante.

```redis
127.0.0.1:6379> SAVE
OK
```

## `BGSAVE`

Inicia un snapshot RDB en segundo plano, sin bloquear el servidor.

```redis
127.0.0.1:6379> BGSAVE
Background saving started
```

## `LASTSAVE`

Muestra el timestamp Unix del último guardado RDB.

```redis
127.0.0.1:6379> LASTSAVE
(integer) 1696802400
```

## `CONFIG GET parámetro`

Recupera valores de configuración del servidor Redis.

```redis
127.0.0.1:6379> CONFIG GET dir
1) "dir"
2) "/data"
```

## `CONFIG SET parámetro valor`

Cambia la configuración en caliente. El cambio es temporal hasta el reinicio, salvo que se persista en el archivo de configuración.

```redis
127.0.0.1:6379> CONFIG SET appendonly yes
OK
```
